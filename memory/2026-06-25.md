## 2026-06-25 — OpenClaw Control UI basePath Fix

- **Time:** 06:12 CDT
- **Status:** ✅ COMPLETE — Control UI moved to `/openclaw/`, PDF links fixed
- **Lesson learned:** Path prefix changes break relative URLs

### What Happened
1. Set `gateway.controlUi.basePath: "/openclaw"` to move Control UI off root path
2. This freed up `/dashboard/` from Control UI script injection
3. BUT: PDF links in dashboard were relative (`/download/...`) which resolved to root path, not `/dashboard/`
4. Fixed by making PDF links absolute with `/dashboard/download/...` prefix

### Critical Lesson: Path Prefixes Cascade
When you change a base path or add a path prefix (like `basePath: "/openclaw"` or reverse proxy `/dashboard/`), ALL relative URLs in the application must be updated to include the correct prefix. This includes:
- API endpoints (`/api/...` → `/dashboard/api/...` or dynamically detected)
- Static assets (`/download/...` → `/dashboard/download/...`)
- Client-side routing
- WebSocket connections

**The fix:** Use `window.location.pathname` to detect the current path prefix at runtime, OR hardcode the full absolute path with the prefix.

### Files Changed
- `~/.openclaw/openclaw.json` — `gateway.controlUi.basePath: "/openclaw"`
- `Systack/content/saos/saos-data/customer-dashboard/index.html` — PDF links updated to `/dashboard/download/...`
- `Systack/content/saos/saos-data/customer-dashboard/api.py` — PDF download routes added

### Next
- Verify all other relative URLs in dashboard work correctly with `/dashboard/` prefix
- Update documentation about path prefix requirements
- Consider making API_BASE and asset URLs dynamically detect prefix to prevent future issues
